VolunteerUP

An app to find & join local volunteer events

VolunteerUP is an app that connects nonprofit organizations with eager volunteers. Users are able to register as volunteers, search for events in their area, and sign up for events as volunteers. Users are also able to register as organizers that administer their nonprofit organization pages, create a calendar of events, and request volunteers.

This app was created during my final group project at LEARN Academy, a rails full-stack development bootcamp in San Diego. The app was created from Rails version 5 and postgresql in the back end.

I was involved in general theme styling and integrating the calendar of events both on the user & organization profile. I added a toggle feature so users can switch between a calendar and map of upcoming events. In the back end, I created the organization and user database relationship so users can have many organizations and organizations can have many users. Feature and integration testing was preformed through RSpec and Capybara.

Wildlife Tracker

An app to track and view animal sightings

The purpose of this app is for park rangers to have a simple way to report wildlife sightings. Rangerâ€™s have the ability to create an animal in the database and add many sightings to that specific animal. An animal can be created by adding the common name, latin name and kingdom. Sighting can be created with the input of a date, time, latitude and longitude.

I utilized a scaffolding command to build a CRUD interface with Ruby on Rails. This way, rangers are able to view, update, edit and destroy the animals and sightings they create.

In the app, I added a jQuery calendar plug-in so users can have a calendar displaying all of a given month's animal sightings. I later included functionality with Ajax, where a user/ranger can go to an animal or sightings page and add additional information without any hard refreshing or redirects of the page.

Weather Watcher

Check to see if you need an unbrella today......

Weather Watcher is a weather app that will take in a location and produce the degrees in F, the location, weather icon, percentage humidity, windspeed and direction. Depending on the weather (ex. sunny, raining, snow, etc.) a different icon will show each time.

I utilized the OpenWeatherMap Api and added in the ability to have the weather updated by current geographic location rather than inputing a zip code or city. If a user's browser isn't supported by geolocation, then it will prompt the user to manually enter in their zip code.
